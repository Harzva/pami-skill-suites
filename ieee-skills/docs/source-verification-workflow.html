Offline-safe by default

CI uses dry-run validation so releases remain reproducible. Maintainers run live checks before public source-refresh releases.

Commands

python scripts/check_official_links_live.py --dry-run
python scripts/check_official_links_live.py --live
python scripts/report_stale_sources.py

Outputs

  • resources/source_refresh_report.json
  • resources/source_refresh_report.yaml
  • resources/stale_source_report.md
  • resources/stale_source_report.json

Important policy

Source health is not the same as rule currency. Even when a URL is healthy, mutable details such as APCs, submission portals, author agreements, license options, templates, page limits, and AI disclosure requirements must be checked on live official pages before submission advice.

Unofficial project. Verify live publisher and journal pages before real submission.
