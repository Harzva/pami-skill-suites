# Limitations Future Work

Publisher family: IEEE

Show intellectual honesty while protecting the main contribution.

## Fixed routines

### Boundary-not-defect

**Sequence:** scope boundary → why boundary exists → evidence that main claim still holds → future extension

**Reviewer risk:** Limitations framed as defects undermine contribution.

### Failure-case taxonomy

**Sequence:** failure mode → trigger condition → observed effect → mitigation/future work

**Reviewer risk:** Unstructured limitations look defensive.

### Ethics-deployment caution

**Sequence:** deployment condition → risk → guardrail → evaluation needed before deployment

**Reviewer risk:** High-impact application papers need responsible caveats.

## Output contract

- limitations table
- future work ladder
- deployment caveat
- reviewer-risk note

## Safety note
These routines are structural best-practice patterns, not official publisher rules and not text to copy from template papers.
