# Writing Patterns for IEEE TIP

## Abstract pattern

Problem → gap → method idea → technical contribution → strongest evidence → implication, with concise expert-reader wording.

## Introduction pattern

Context → limitation of prior art → exact technical gap → proposed solution → contributions → evidence preview.

## Related work pattern

Group prior work by technical families, explain their limitations without strawman phrasing, and end with a precise positioning paragraph that says what this paper changes.

## Method pattern

Define notation before use, motivate each design choice, connect equations/algorithms to implementation, and state assumptions openly.

## Experiments pattern

State datasets, splits, metrics, baselines, training/testing details, hardware/runtime when relevant, statistical protocol when relevant, and what each table or figure proves.

## Discussion and limitation pattern

Separate confirmed findings, plausible interpretations, known weaknesses, external validity limits, and future work.
