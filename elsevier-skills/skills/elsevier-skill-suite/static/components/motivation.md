# Motivation Component Module

## Purpose

Organize the paper motivation as a concrete chain from pain point to technical gap to method necessity.

## Fixed routines

- Pain-point → limitation of existing methods → missing mechanism → proposed design principle.
- Toy contrast / motivation figure: old paradigm → failure mode → new insight → method sketch.
- Gap taxonomy: solved / unsolved / this paper / deliberately out of scope.

## Output contract

- Motivation ladder.
- Gap table.
- Main-figure or motivation-figure brief.
- Overclaim guardrails.

## Pairing modules

- problem-formulation
- introduction
- related-work
- figure
- caption

## Safety

This module provides repository best-practice structure, not publisher official policy.
