# Skill-suite router source notes

This release references official skill-system and GitHub Pages documentation to justify the product architecture:

- skills as folders with `SKILL.md` plus optional references/scripts/assets;
- progressive disclosure: metadata first, full instructions and resources only when needed;
- description-driven implicit selection;
- static GitHub Pages site under `docs/` or via GitHub Actions;
- custom GitHub Pages workflows for static site deployment.

These notes describe product architecture only. Publisher/journal manuscript rules still require live official-source verification.
