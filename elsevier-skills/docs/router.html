v0.5.0 router architecture

Skill-in-skill without hiding the skills.

The source-of-truth layout keeps skills flat and independently installable. Router skills orchestrate them through registries. The compact suite is generated separately for one-skill installation.

Architecture

skills/
├── elsevier-skill-suite/          # suite-level router
├── elsevier-journal-router/  # journal adapter router
├── elsevier-component-router/# manuscript component router
├── elsevier-submission-router/# submission workflow router
├── <component-skill>/
└── <journal-adapter>/

resources/
├── skill_registry.json
├── journal_registry.json
├── component_registry.json
├── submission_registry.json
└── routing_rules.json

dist/compact-suite/
└── elsevier-skill-suite/          # one-skill distribution

Why flat source layout?

Flat skills stay directly discoverable, installable, and testable. The router can coordinate them, but each journal adapter and manuscript component remains a real skill. Nested modules are used only in the generated compact distribution.

Route output contract

FieldPurpose
